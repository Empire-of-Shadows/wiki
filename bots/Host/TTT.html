Tic‑Tac‑Toe — User Guide

TL;DR

  • Start in the TTT lobby with /tictactoe start; choose grid size (3×3 or 5×5) and options.
  • Each match gets its own channel plus a discussion thread (Public or Private).
  • Play by clicking board buttons — X goes first unless changed.
  • Game over posts results, updates the TTT leaderboard, and cleans up shortly after.

Quick start #

  1. Go to the designated TTT lobby channel for your server.
  2. Run /tictactoe start and choose:
    • Difficulty: Easy (3×3) or Medium (5×5)
    • Opponent (optional): Invite a user or leave open
    • Who starts (optional): Me, Opponent, or Random
    • Visibility (optional): Public or Private discussion thread
  3. The bot creates a new game channel and posts a lobby message. If you invited someone, only they can join; otherwise anyone can use the Join button.
  4. Once two players are in, the bot posts the clickable board and starts the match.

Playing the game #

  • The board is a grid of buttons. Click to place your mark:
    • Player 1: X
    • Player 2: O
  • The embed shows whose turn it is; invalid moves are blocked with an ephemeral notice.
  • Use the linked thread for chat, strategies, or banter to keep the game channel clean.

Discussion thread & channel rules #

  • Game channel is read‑only except for gameplay actions to keep the board tidy.
  • A discussion thread is created automatically:
    • Public or Private per your start options.
    • If Private and you specified an opponent, they’re added automatically.

Ending the game #

  • Win: Three in a row along rows, columns, or diagonals.
  • Tie: Board fills without a winner.
  • When a match ends:
    • A final “Game Over” embed is posted in the game channel.
    • Results are posted to the configured win feed channel.
    • The channel is scheduled for cleanup and deletion shortly after (≈30s).

Inactivity cleanup #

  • Idle games may be cleaned up automatically to keep the server tidy.

Canceling a game #

  • Use /tictactoe cancel in the game channel.
  • Allowed if you are one of the two players or a moderator with Manage Channels.

Leaderboard (TTT) #

View rankings based on persistent Tic‑Tac‑Toe results.

Command

  • /leaderboard tictactoe

What you’ll see

  • Mention — W wins | L losses | T ties | WR win rate%.
  • Pagination with Previous/Next after 10 entries.
  • Graceful empty state if no results yet.

Options

  • top: How many users to display (1–50). Default: 10.
  • sort: Autocomplete list, e.g. wins_desc, losses_asc, ties_desc, win_rate_desc, net_wins_desc, etc.

Rate limits

  • Light per‑guild cooldown (~3 seconds) to prevent spam.

Tips & common questions #

  • Can’t start a game? Use the designated TTT lobby channel.
  • Board didn’t update? Make sure it’s your turn and the cell is empty.
  • Private discussion? Choose Private for the thread and invite an opponent.
  • Cleanup timing? Channels auto‑delete shortly after a match ends; save anything you need from the thread.

Phased Roadmap #

Phase 1 — Foundation & QoL (ongoing)

  • Rematch flow — one-click rematch for both players with 60s accept window, auto-new game channel, preserves last options unless changed.
  • Invites with expiration — lobby invites expire after 5–10 minutes; expired invites show disabled Join button and hint to restart.
  • Match history (last 10) — per-user lightweight history: opponent, result, grid size, duration, timestamp; accessible via ephemeral UI.
  • Blitz mode (per-move timer) — configurable 10/15/30s per move; visible timer on embed; auto-forfeit on timeout after 1 grace warning.
  • Board UX — clearer turn indicator, disabled cells after tap, inline invalid-move reasons, compact mobile layout for 5×5.
  • Leaderboard polish — pagination arrows + page indicator, remember last sort, tie-breaks: win rate → net wins → recent form.
  • Error handling — friendly ephemeral errors, retry on Discord API rate limits, idempotent button handlers.
  • Metrics & logging — match start/end, abandons, average turns, move latency, timeout rate, invite acceptance; redaction for PII.
  • Moderation tools — quick Cancel from thread by mods, reason codes, and short audit log entry.
  • Cleanup — automatic archive/delete of game channels ~30s post-game, safe-guard to avoid deleting active threads.

Phase 2 — Party Mode Beta (ongoing)

  • Party lobby — create/join/leave commands; party owner transfer on leave; max party size controls.
  • Modes — King of the Hill (winner stays) and Hot‑Seat (rotating pairs) with configurable round length and win target.
  • Lightweight scoreboard — wins, streaks, plus “on-fire” indicator; resets when party ends; export summary to thread.
  • Gimmicks toggle — optional variants (e.g., center lock, first-move swap) with per-party enable/disable and clear tags on board.
  • Party thread — dedicated chat thread for banter and coordination; owner-only config buttons.
  • Queue management — join order visible; AFK detection with skip after 30s; rejoin grace window.
  • Admin controls — soft rate limits on party creation, kill switch for mode rollout, per-guild caps.
  • Anti-abuse — duplicate-opponent cooldown in parties, match length minimum for stat credit, bot/alt filters.
  • Observability — party retention, average rounds, mode popularity, drop-off points.

Phase 3 — Tournament MVP (ongoing)

  • Bracket creation (4–64) — host picks size, single-elim, optional 3rd-place match; auto seeding or manual.
  • Seeding — random, leaderboard-based, or custom order; lock-in preview before publish.
  • Match orchestration — on-round start, auto-create match channels with linked threads; winners progress automatically.
  • Reporting — winner button for players; moderator override (replace, forfeit) with reason logging.
  • Timers — check-in window and per-round deadlines; auto-DQ on repeated no-shows with notice.
  • Substitutions — host can replace a player before first move; bracket integrity validated.
  • Spectator view — read-only bracket message with live updates, pinned in the hub channel.
  • Persistence — resume brackets after bot restart; backfill results if match finished while offline.
  • Exports — bracket summary posted at end: podium, upset count, fastest win, most moves.

Phase 4 — Competitive Loop (ongoing)

  • Ranked queue — solo queue with role-based MMR visibility; optional warmup unranked toggle.
  • ELO — Glicko-lite or standard ELO with K-factor based on provisional status and opponent delta.
  • Placement — first 10 ranked matches provisional; larger MMR swings; visible “Provisional” badge.
  • Seasons — scheduled resets; season leaderboard and badges; carryover of lifetime stats.
  • Anti-farm — cooldown between repeat matchups, IP/device pattern flags (privacy-safe), forfeit penalties.
  • Decay — gentle MMR decay after inactivity; resumes after 2–3 weeks idle.
  • Profiles — per-user page: ELO, rank tier, season badges, rival list, streaks, last 10 ranked.
  • Match quality — estimated win probability and expected MMR delta preview in ranked queue.
  • Notifications — opt-in pings for rank-ups, top‑10 entries, and end-of-season rewards.

Phase 5 — Delight & Scale (ongoing)

  • Achievements — milestones (first win, streak 5, comeback), mode-specific badges; earnable cosmetics currency.
  • Cosmetics — profile frames, match intro flair, win animations in embeds; all non-paywalled, earnable.
  • Replays & highlights — compact move list with timestamps; auto-generate highlight summary for notable matches.
  • Advanced tournaments — double-elim and pools; simple TO tools for scheduling and substitutions.
  • Localization — i18n strings, pluralization rules, RTL support; community translation pipeline.
  • Accessibility — high-contrast board theme, screen-reader labels for cells, reduced-motion mode.
  • Performance — batched updates to reduce API calls; shard-aware matchmaking; cost monitors and alerts.
  • Analytics — cohort retention, conversion to ranked, party → tournament funnels; privacy-first aggregation.
